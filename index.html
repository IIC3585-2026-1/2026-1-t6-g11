Web Components.

Siete componentes construidos solo con los estándares de la plataforma: HTML templates, Custom Elements y Shadow DOM. Sin frameworks. Sin build. Reutilizables en cualquier página.

Cada demo de abajo vive dentro de un borde punteado: ese es el shadow boundary, la frontera que aísla los estilos y el markup interno del componente del resto de la página.

01

Campo numérico

Un input de número con botones para incrementar y decrementar. Emite un evento change cada vez que el valor cambia, ya sea por los botones o escribiendo directo.

Número: Temperatura:

Escuchando el evento desde la página → Número: 0 · Temperatura: 42

<campo-numerico value="0">Número:</campo-numerico>
<campo-numerico value="42">Temperatura:</campo-numerico>

Encapsulación: los botones y el input viven en el Shadow DOM, así que sus estilos no chocan con los de la página. El único canal de comunicación hacia afuera es el evento change con detail.value.

02

Slider

Un control deslizante con burbuja de valor y etiquetas posicionadas por porcentaje. Útil para rangos como temperatura, volumen o brillo.

0℃ 25℃ 50℃ 75℃ 100℃

Valor actual → 50

<mi-slider min="0" max="100" value="50" step="5">
  <mi-slider-label>0℃</mi-slider-label>
  <mi-slider-label position="50">50℃</mi-slider-label>
  <mi-slider-label position="100">100℃</mi-slider-label>
</mi-slider>

Encapsulación: las etiquetas se pasan como hijos (<mi-slider-label>) y el componente las reposiciona internamente según su atributo position. La página no necesita saber cómo se dibuja la barra por dentro.

03

Switch

Un interruptor on/off con mensaje configurable según el estado. El texto de cada estado se inyecta por slot, así que el mismo componente sirve para "On/Off", "Activado/Desactivado", etc.

On Off Activado Desactivado 
<mi-switch>
  <span slot="checked-message">On</span>
  <span slot="unchecked-message">Off</span>
  <label>Captions:</label>
</mi-switch>

<mi-switch checked> ... </mi-switch>  <!-- mismo componente, otro estado -->

Encapsulación: el atributo checked refleja el estado y se observa con observedAttributes; los slots dejan que la página personalice el texto sin tocar el interior del componente.

04

Breadcrumb

Migas de pan para navegación: muestra la ruta jerárquica donde está el usuario. Los ítems con href son enlaces; el último, sin href, es la ubicación actual.

Home Next Level Down Current Location 
<mi-breadcrumb>
  <mi-breadcrumb-item href="#">Home</mi-breadcrumb-item>
  <mi-breadcrumb-item href="#">Next Level Down</mi-breadcrumb-item>
  <mi-breadcrumb-item>Current Location</mi-breadcrumb-item>
</mi-breadcrumb>

Encapsulación: el separador / se dibuja con CSS (::after) dentro del Shadow DOM, no como texto en el markup. El componente decide su propia tipografía y espaciado sin que la página interfiera.

05

Acordeón

Paneles colapsables: cada ítem muestra un encabezado y oculta su contenido hasta que se hace clic. Ideal para FAQs o secciones largas. Permite varios ítems abiertos a la vez.

Item 1
Panel one content
 Item 2
Panel 2 content
 Item 3
Panel 3 content
 
<mi-accordion>
  <mi-accordion-item>
    <span slot="heading">Item 1</span>
    <div>Panel one content</div>
  </mi-accordion-item>
  ...
</mi-accordion>

Encapsulación: la animación de apertura (max-height + rotación del ícono) está toda en el Shadow DOM y se dispara con el atributo open. El contenido del panel se pasa por slot, así cada ítem lo define la página.

06

Tarjeta

Una tarjeta contenedora cuyo tamaño se controla desde fuera con CSS, sin tocar el componente. Demuestra cómo personalizar un componente encapsulado.

Custom size using CSS Default size 
<mi-card class="custom">Custom size using CSS</mi-card>
<mi-card>Default size</mi-card>

/* en la hoja de estilos de la página */
mi-card.custom {
  --card-width: 150px;
  --card-height: 100px;
}

Encapsulación: las CSS custom properties (--card-width, --card-height) son el único canal de estilo que cruza el shadow boundary. La página las define con var() y el componente las consume por dentro — todo lo demás queda aislado.

07

Horizontal scroll

Un contenedor que organiza a sus hijos en una fila desplazable horizontalmente. Sirve para carruseles de tarjetas, galerías o listas que no caben en el ancho.

Card 1 Card 2 Card 3 Card 4 Card 5 
<mi-horizontal-scroll>
  <mi-card>Card 1</mi-card>
  <mi-card>Card 2</mi-card>
  ...
</mi-horizontal-scroll>

Encapsulación: usa ::slotted(*) para evitar que las tarjetas se encojan, aplicando estilo a los hijos proyectados sin sacarlos de su lugar. Composición de componentes: tarjetas dentro de un scroll, cada uno ignorante del otro.
